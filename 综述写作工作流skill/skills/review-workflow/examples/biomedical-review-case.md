# 生物医学综述案例抽象

本文件不是原项目复制，而是对一个真实综述项目工作流的抽象总结，供 skill 使用者理解如何把案例工程转成可复用方法。

## 案例特点

- 主题：大语言模型在生物医学文本挖掘与知识发现中的应用综述
- 类型：中文课程综述
- 过程要求：全流程留痕
- 风险要求：不得混入虚假或不相干文献
- 目标要求：最终有效参考文献不少于 40 篇

## 案例工作流

1. 明确题目边界与最低证据目标
2. 搭建候选文献池与筛选规则
3. 形成结构化证据提取表、事实核查记录与阅读笔记
4. 依据证据而不是直觉生成提纲
5. 从初稿到修改稿逐步提升“证据可见度”和“比较性”
6. 进行模拟审稿与逐条回复
7. 统一引文编号与 Word 成稿输出

## 从案例中提炼出的关键经验

### 1. 证据层先行
没有足够的结构化证据，不应急于写正文。

### 2. 证据必须在正文中可见
不能只在表格里有证据，正文却只写“有研究显示”。

### 3. 薄弱方向也要正面交代
不能因为证据少，就在综述里完全略过 RE / 事件抽取 / PICO 等方向。

### 4. “知识发现”必须严格界定
支撑层任务与知识发现层任务必须区分，否则题目与正文会错位。

### 5. 定稿不是简单润色
定稿阶段往往还包括：
- 引文清洗
- 参考文献排序
- 图表位置整理
- 审稿痕迹删除
- Word 输出规范化

## skill 使用建议

将这个案例当作“流程样本”，而不是直接复刻内容。复用时应替换：
- 主题
- 数据库
- 标签体系细节
- 章节重点
- 引文映射

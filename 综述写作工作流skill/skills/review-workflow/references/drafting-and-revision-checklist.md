# 起草与修订检查表

## 起草阶段重点

1. 先保证关键判断有明确引文锚点。
2. 先把薄弱方向写出真实边界，不要回避。
3. 技术路径段落要落到具体方法实例，而不是只有抽象判断。
4. 对核心概念（如“知识发现”）集中定义，避免多处重复展开。
5. 未来方向要尽量写成可执行设计，而不是愿望清单。

## 修订阶段重点

### 应优先修的问题
- 证据层“隐身”
- 章节之间重复论证
- 代表性工作缺少横向比较
- 过于绝对的措辞
- 图表不足
- 正文与题目不完全匹配

### 推荐修订顺序
1. 补显性引文锚点
2. 补弱势子方向最小可讨论段落
3. 强化比较式写法
4. 收缩重复概念段
5. 具象化未来方向
6. 再做语言精修

## 学术表达提醒

- 用“现有证据提示”“更稳妥的判断是”“现阶段更接近于”替代过度绝对化表达
- 区分“潜力”“概念验证”“较成熟方向”“主干证据”

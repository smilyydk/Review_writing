# 标签体系模板

## 标签原则

每篇文献可有多个标签，但应优先选择最能体现其贡献的标签。标签应服务于：
- 证据抽取
- 提纲分节
- 比较表构建
- 证据强弱判断

## 推荐维度

### 1. 任务维度
- 文献理解与摘要生成
- 命名实体识别（NER）
- 关系抽取（RE）
- 事件抽取
- 问答（QA）
- 检索增强与知识检索
- 知识图谱构建/利用
- 知识发现
- 假设生成
- agent / 工具调用 / 数据库协同
- 可靠性与评测

### 2. 方法维度
- 通用 LLM
- 领域 LLM
- 指令微调
- 提示工程
- RAG
- GraphRAG / KG-grounded
- agent
- 外部工具调用
- 外部数据库接入
- benchmark / evaluation

### 3. 数据维度
- 文献摘要
- 论文全文
- 临床文本
- 知识库 / 数据库文本
- benchmark 数据集
- 混合知识源

### 4. 证据类型维度
- 方法论文
- 应用研究
- benchmark 研究
- 综述论文
- case study
- 观点/评论
- 数据集/语料论文
- 工具/系统论文

## 提醒

- 标签要支持后续表格统计与章节映射
- “知识发现”标签应慎用，不能把所有检索增强工作都标成知识发现

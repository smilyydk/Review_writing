# 可脚本化扩展建议

当前 skill 主体以流程与模板为主。若后续要增强自动化，可优先扩展以下脚本：

1. 候选文献表初始化脚本
2. 证据提取表模板生成脚本
3. 阅读笔记批量初始化脚本
4. 引文映射检查脚本
5. Markdown -> 编号引文 -> Word 导出脚本

## 脚本设计原则

- 与 skill 主文档解耦
- 输入输出路径清楚
- 失败时显式报错，不静默跳过
- 适合放到 GitHub 仓库中独立维护

## 推荐后续动作

- 将本 skill 与你当前项目中的 `build_v4_and_docx.py` 这类脚本建立松耦合参考
- 后续可以把通用脚本迁移到本仓库 `scripts/` 目录下
- 如果要做更标准的插件分发，可继续补充测试样例与触发优化
